# Bloom ‚Äî a sellable Flutter pregnancy companion, generated end-to-end

A Converge showcase. Edit `idea.md`, run one script, and the playbook produces an Android-buildable Flutter app: 11 screens, Riverpod state, GoRouter navigation, 60 generated SVG assets, widget tests ‚Äî all gated by real toolchain checks (`flutter pub get`, `dart analyze`, `flutter build apk --debug`, `flutter test`).

## What this demonstrates

- **Multi-phase Converge DAG** ‚Äî 8 sequential tasks turn `idea.md` into a buildable app.
- **Dynamic spawn from body** ‚Äî `seed: { mode: cli }` tasks read manifests at runtime and emit `converge spawn template ‚Ä¶` for every screen / overlay / asset / test. No hand-written copies.
- **Swappable image backend** ‚Äî `svg-illustration-generation` skill ships a MiniMax primary and a stub for CI; switch via `backends/ACTIVE` or `IMAGE_BACKEND` env var.
- **Real production goal gates** ‚Äî the playbook only converges when `flutter build apk --debug` and `flutter test` succeed, no `onPressed: null` survives, no `Lorem|TODO|FIXME` lingers, and every `navigations.json` element reports `status: wired`.

## Prerequisites

| Tool | Why |
| --- | --- |
| Flutter SDK 3.24+ | `flutter pub get`, `dart analyze`, `flutter build apk --debug`, `flutter test` |
| `converge` CLI | playbook execution (`npm i -g @openplaybooks/converge` or build from monorepo) |
| `jq` | task bodies parse `.stitch/screens.json` |
| Android SDK + cmdline-tools | required for `flutter build apk --debug` |

Environment:

| Variable | Required | Purpose |
| --- | --- | --- |
| `MINIMAX_API_KEY` | yes | LLM provider (via Anthropic-compat endpoint) + image API auth |
| `MINIMAX_IMAGE_ENDPOINT` | yes when `IMAGE_BACKEND=minimax` | URL of the MiniMax text-to-image endpoint |
| `MINIMAX_IMAGE_MODEL` | no | image model id, defaults to `image-01` |
| `IMAGE_BACKEND` | no | `minimax` (default) or `stub` |

## Quick start

```bash
export MINIMAX_API_KEY=‚Ä¶
export MINIMAX_IMAGE_ENDPOINT=https://api.minimax.io/v1/image/generate  # or your endpoint
./scripts/clean.sh        # wipe generated artifacts; keep inputs
./scripts/run.sh          # ~30‚Äì60 min end-to-end
```

For a free smoke test (no image API spend) use the stub backend:

```bash
IMAGE_BACKEND=stub ./scripts/clean.sh && ./scripts/run.sh
```

The stub writes valid grey-box SVG placeholders so every downstream check (sizes, count, wire-up, build) still passes ‚Äî useful for CI and verifying the playbook itself.

## Customization

- **Change the app idea**: edit `idea.md`. The playbook re-derives PRD ‚Üí UX ‚Üí screens ‚Üí code on next run.
- **Change the asset catalog**: edit the inline arrays in `.converge/playbooks/default/tasks/08-generate-assets/TASK.md` (40 weeks, 5 empty states, 15 icons). Counts in `playbook.yml`'s `assets-generated` goal must match.
- **Swap image backend**: edit `.converge/skills/svg-illustration-generation/backends/ACTIVE` (one line: `minimax` or `stub`), or set `IMAGE_BACKEND=‚Ä¶` per invocation. Add a new backend by dropping a `backends/<name>/generate.js` file that exports a default function matching the contract in `SKILL.md`.
- **Bump the LLM model**: edit `.converge/project.yml`. Defaults to `MiniMax-M2.7` via the Anthropic-compat endpoint.

## Troubleshooting

| Symptom | Cause | Fix |
| --- | --- | --- |
| `MINIMAX_API_KEY must be exported` | env not set | `export MINIMAX_API_KEY=‚Ä¶` before `./scripts/run.sh` |
| Task 08 fails immediately with "MINIMAX_IMAGE_ENDPOINT is not set" | minimax backend can't reach API | export the endpoint or `IMAGE_BACKEND=stub` |
| `flutter build apk --debug` fails with Android SDK error | cmdline-tools missing | `sdkmanager "platform-tools" "build-tools;34.0.0" "platforms;android-34"` |
| `flutter test` fails on first run after rebuild | stale `.dart_tool/` | `./scripts/clean.sh` then re-run |
| `dart analyze` warns about unused `await` in a generated test | the generator missed a callback; task 09's check will fail and the orchestrator retries | usually self-heals on retry; otherwise inspect `test/screens/<name>_test.dart` |
